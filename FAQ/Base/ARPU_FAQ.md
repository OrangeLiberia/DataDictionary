# 📊 ARPU Query – FAQ & Documentation

## 📌 Overview

This document explains the SQL query used to retrieve **ARPU-related data** for a specific subscriber on a given date.

---

##  What is ARPU?

**ARPU (Average Revenue Per User)** is a key metric used in telecom and business analytics to measure the **average revenue generated per customer** over a specific period.

---

## What is the purpose of this query?

This query retrieves:

- A specific **subscriber (MSISDN)**  
- Their associated **RateBill value**  
- For a defined **date (March 1, 2026)**  

👉 It is typically used for **ARPU validation, reporting, or analysis**.

---

##  What tables are used?

### 🔹 1. SubscribersGroups (`BI.in.SubscribersGroups`)
- Contains subscriber-level data  
- Includes:
  - `MSISDN` (customer number)  
  - `RateBill` (revenue or billing value)  
  - `IDDimDate` (date reference key)

---

### 🔹 2. DimDate (`bi.fwk.DimDate`)
- Date dimension table  
- Used to translate `IDDimDate` into a readable date (`dt`)

---

## How are the tables joined?

```sql
INNER JOIN bi.[fwk].[DimDate] d
ON S.[IDDimDate] = d.id

👉 This join links subscriber data to the actual calendar date.

 What filters are applied?
WHERE d.dt = '2026-03-01' 
  AND [MSISDN] = '777777732'
Filters data for:
📅 Specific date: March 1, 2026
📱 Specific subscriber: 777777732
❓ What fields are returned?
Field Name	Description
MSISDN	Subscriber phone number
RateBill	Revenue or billing amount (used for ARPU calculation)

How does this relate to ARPU?
RateBill represents the revenue generated by the user
ARPU can be calculated by:
Aggregating RateBill across users
Dividing total revenue by number of users

📎 SQL Query
SELECT 
    [MSISDN],
    [RateBill]
FROM [BI].[in].[SubscribersGroups] S
INNER JOIN bi.[fwk].[DimDate] d
    ON S.[IDDimDate] = d.id
WHERE d.dt = '2026-03-01' 
  AND [MSISDN] = '777777732';

Can this query be extended?

Yes, you can:

Remove MSISDN filter to get all users
Aggregate for full ARPU calculation:
SELECT 
    AVG(RateBill) AS ARPU
FROM [BI].[in].[SubscribersGroups] S
INNER JOIN bi.[fwk].[DimDate] d
    ON S.[IDDimDate] = d.id
WHERE d.dt = '2026-03-01';
Group by segments (e.g., region, plan)
